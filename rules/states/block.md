# BLOCK 检查表

## 6 个触发模式

| # | 模式 | 信号 |
|---|------|------|
| 1 | **前提被推翻** | "实际没有这个字段" / "函数不存在" / "路由没注册" |
| 2 | **需要外部资源** | 缺权限/凭证/API key / 服务不可达 |
| 3 | **分叉路口** | 决策影响大（重构 vs 绕过 / 删旧 vs 兼容），不能代选 |
| 4 | **连续失败** | 同一验证 3 次修复后仍不过 |
| 5 | **需要审批** | 触发删除/迁移/强推/付费 API 等不可逆操作 |
| 6 | **无测试推进** | 代码改动完成但没写测试 |

## 必须记录

- [ ] 阻塞原因（具体，不是"过不了"）
- [ ] 已尝试的动作
- [ ] 需要什么外部输入或权限
- [ ] 记录到 Documentation.md §5 或 §9

## 模式 6 特殊处理

BLOCK 后不是等用户指令 → 立即回到 `EXECUTE` 补测试。
仅可豁免：纯配置/纯文档/纯 UI 样式。

## 模式 3（分叉路口）

不是完全卡死。当前任务暂停，但可切到其他能做的工作。
用 Help Gradient "判断" 级别汇报。
