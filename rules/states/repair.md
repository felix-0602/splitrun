# REPAIR 检查表

## 修复协议

1. [ ] 定位根因（不靠猜，看错误输出 file:line）
2. [ ] 最小修复（只改导致失败的代码）
3. [ ] 重新 VALIDATE

## 报错反馈格式

```
**现象**：[具体表现，附 file:line 或命令输出]
**根因**：[一句话——什么导致的]
**修复**：[改了什么]
**剩余风险**：[改了之后要注意什么，或"无"]
```
如果试过其他路径：**尝试过但否决**：[方案 + 否决原因]

## 卡壳恢复（同一问题修 2 次仍失败）

- [ ] 不继续硬修
- [ ] 废弃当前方案
- [ ] Snapshot：已发现的问题 + 尝试过的路径
- [ ] 收窄范围，重新定义最小可验证步骤
- [ ] 重试

## 上限

- [ ] 最多连续修复 3 轮
- [ ] 3 轮后仍失败 → BLOCK（附带 snapshot）
- [ ] **不得**通过删除关键断言、跳过真实场景、迎合 evaluator 来伪修复

## 工具

`Skill(systematic-debugging)` — 根因定位优先
