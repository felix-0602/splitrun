# transition/legal: 状态转移合法性

**防止什么危害**：模型跳过中间状态，从 READ_CONTEXT 直接跳到 EXECUTE。

**为什么存在**：状态机是 DEEPSHIP 的骨架。每个状态有明确的进入条件和退出条件。转移表定义了哪些路径合法——不合法的转移意味着中间步骤被跳过了。

**不遵守的后果**：
- READ_CONTEXT → EXECUTE（跳过 MAP_REALITY, PLAN_STEP）→ 代码没有勘察依据
- VALIDATE → COMPLETE（跳过 RECORD, ADVANCE）→ 没有记录、没有归档
- 任意状态 → COMPLETE → 丢失所有中间状态的可追溯性

**合法转移图**：
```
READ_CONTEXT → CLARIFY_INTENT | MAP_REALITY
CLARIFY_INTENT → MAP_REALITY | BLOCK
MAP_REALITY → SELECT_MILESTONE | BLOCK
SELECT_MILESTONE → PLAN_STEP | BLOCK
PLAN_STEP → EXECUTE
EXECUTE → VALIDATE
VALIDATE → RECORD | REPAIR
REPAIR → VALIDATE | BLOCK (< 3次)
RECORD → ADVANCE
ADVANCE → READ_CONTEXT | COMPLETE
BLOCK → READ_CONTEXT
COMPLETE → READ_CONTEXT
```
