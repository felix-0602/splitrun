# 安全约束（静态层）

> **在所有状态下适用。** EXECUTE 状态必须加载。受益于 prompt caching。
> 完整原文：`implement/safety.md`

## 每次改动前自检 (C.1)

- [ ] 涉及用户输入处理？→ 加了校验吗？
- [ ] 涉及文件系统操作？→ 防路径遍历了吗？
- [ ] 涉及数据库查询？→ 参数化了吗？
- [ ] 涉及 API 调用？→ 有 timeout 吗？
- [ ] 涉及敏感数据？→ 没硬编码吧？
- [ ] 破坏了什么契约？→ 类型/文档/测试同步更新了吗？
- [ ] 改动扩到无关文件了吗？→ 只有契约同步类不在此列
- [ ] 本轮调了 code-reviewer 吗？→ **必调，不跳**

## Diff 边界 (C.2)

- 改一个 bug 就别顺手"优化"旁边的代码
- 加一个功能就别重命名无关变量
- 发现需要改的旁边代码 → 记到 Documentation.md，收手

## 契约同步 ≠ 扩 Diff (C.2.1)

改了函数签名/API 响应/env/配置/启动命令/业务逻辑/数据 schema →
对应的类型定义、文档、测试必须在同一 commit 同步更新。

判断标准：新人按旧文档操作会不会跑不起来？会 = 没改完。

## 安全审查触发词 (C.5)

看到以下场景 → **立即调 `Agent(security-reviewer)`**：
auth / 用户输入 / 数据库查询 / 文件系统 / 外部 API / 加密 / 支付

## 不可逆操作 (C.3)

以下必须先向用户确认：
删除文件/目录、DROP TABLE、force-push、CI/CD 变更、付费 API

## Secret 检查（提交前）

- [ ] 无硬编码密钥
- [ ] 无连接字符串含凭据
- [ ] .env 等在 .gitignore
- [ ] 日志不打印 token/password/PII
